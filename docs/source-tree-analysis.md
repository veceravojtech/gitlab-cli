## Critical Directories

### `/cmd/gitlab-cli/`
**Purpose**: Application entry point
**Entry Point**: `main.go` - Minimal bootstrap that calls `cli.Execute()`

### `/internal/cli/`
**Purpose**: Command layer - all Cobra command definitions
**Key Files**:
- `root.go` - Root command, global flags (--config, --verbose)
- `mr.go` - Core MR operations (848 lines, most complex file)
- `activity.go` - Activity tracking with multiple output formats

**Commands Registered**:
- `mr list|show|rebase|merge|create|label|auto-merge|reviewer`
- `activity list`
- `project list`
- `label list`
- `user list`
- `config init|show`
- `version`

### `/internal/config/`
**Purpose**: Configuration management using Viper
**Features**:
- YAML file config (`~/.gitlab-cli.yaml`)
- Environment variable support (`GITLAB_URL`, `GITLAB_TOKEN`)
- Sensible defaults (timeouts, retry counts)

### `/internal/gitlab/`
**Purpose**: GitLab API v4 client
**Pattern**: Repository pattern - each resource type has its own file
**Key Types**: `MergeRequest`, `User`, `Pipeline`, `Project`, `Label`, `Event`

### `/internal/progress/`
**Purpose**: Terminal UI for long-running operations
**Features**: Animated wait indicators, elapsed time tracking, colored output
